## P0 — Correctness blockers. Fix BEFORE any further API spend (they corrupt data at collection/analysis time)

### P0.1 Unify the evidence token space (review §8.2 — the most important fix in this plan)
R evidence is spaCy **lemmas** ([`parser.py:207`](src/parsing/parser.py)); H/RO/CF are **surface forms**; with `use_lemmatization: false` they never reconcile → H–R Jaccard = 0.0 on morphologically inflected overlaps (proven in the smoke run). Every ECS number ever reported is deflated by this.

**Decision required (pick one canonical space):**
- **(a) Lemma space (recommended):** set `use_lemmatization: true` so `normalize_tokens` lemma-collapses ALL four strategies' sets. Cheapest, symmetric, matches the anchoring criterion already in use.
- (b) Surface space: make `parse_rationale` return the matched **input surface forms** its anchor already located (requires threading the match back out of `is_anchored`).
- (c) Comparison-time equivalence: make Jaccard/overlap count tokens as equal when their WordNet anchor-lemma sets intersect (keeps stored sets as-is; touches every metric call site).

**Regardless of choice, add fixture tests that pin the scientific invariant:**
- Two evidence sets differing only by inflection (`{moved, scenes}` vs `{move, scene}`) → Jaccard 1.0.
- Synthetic all-agree instance → ECS = 1.0; all-disjoint → ECS = 0.0.
- These are the tests 501 green tests didn't include and would have caught §8.2/§8.3 immediately.

### P0.2 Normalize CF evidence like everything else (review §8.3)
[`run_experiment.py:420`](scripts/run_experiment.py): pass `from_tokens` through `normalizer.normalize_tokens(...)` before storing as the CF evidence set (keep the raw edited-token set in a separate field for the erasure pass / minimality accounting). Also fixes the Monte-Carlo null mismatch: the null draws from the content vocabulary, so the compared sets must live in it.

### P0.3 Whitelist contracted negations as polarity (review §8.4)
Extend `POLARITY_WORDS` in **both** [`parser.py:30`](src/parsing/parser.py) and [`normalizer.py:31`](src/normalization/normalizer.py) with contracted negations and their tokenization variants: `n't, not, don't, doesn't, didn't, isn't, aren't, wasn't, weren't, won't, wouldn't, can't, cannot, couldn't, shouldn't, shan't, mustn't, needn't, hasn't, haven't, hadn't, ain't` (plus apostrophe-stripped forms `dont, isnt, shouldnt, …` since `pre_normalize` strips punctuation). Symmetric across H/R/CF/RO. This is load-bearing for MNLI, where negation IS the evidence.

### P0.4 Per-model erasure engines (review §8.1 — blocks Headline #1)
[`run_validity_tests.py`](scripts/run_validity_tests.py): group `instance_results.jsonl` records by their `model` field; build one `InferenceEngine` per model id (resolve display name → id via `config.models`); re-classify each record with **its own** model. Fail loudly on records whose model is not in the current config (don't silently fall back to `models[0]`). Aggregate per model as well as overall.

### P0.5 Fix the wrong-prediction report rendering (review §8.5)
[`data_models.py:773`](src/utils/data_models.py): `if not r.correct:` must not print "SKIPPED … no explanation strategies elicited, no ECS computed" — that's false since D5. Render full details with a `⚠ wrong prediction — included per D5 (incorrect-prediction stratum)` banner. Reserve SKIPPED for the genuinely short-circuited unparseable-classification case (detect via `n_valid_strategies == 0 and not raw_highlighting`, or better: store an explicit `skipped_reason` field).

### P0.6 Snapshot the prompts actually executed (review §1.3 re-check)
Root cause: [`create_prompt_map`](scripts/run_experiment.py) derives `*_explain.txt` (and dataset variants) from `strategy.prompt_file` by string replacement; `config_snapshot.yaml` serializes the raw config value. Fix both ends:
- Point `config/experiment.yaml`'s `prompt_file` entries at the **executed** files (`prompts/highlighting_explain.txt`, …).
- Write the fully **resolved prompt map** (strategy → file path + SHA-256 of contents) into the run dir (e.g. `prompt_manifest.json`). The report already embeds prompt texts; the manifest makes the provenance machine-checkable.
- Delete the dead `prompts[strategy.id] = prompt_text` load ([`run_experiment.py:101`](scripts/run_experiment.py)) — written, never read.

### P0.7 Normalize both rank lists identically (review §8.6b)
[`run_experiment.py:498–507`](scripts/run_experiment.py): `h_ordered` keeps raw-case salience keys; `ro_ranked` is normalized. Apply the same normalization (case + punctuation + P0.1's canonical space) to `h_ordered` before `assign_implicit_ranks`. Expect Kendall τ / RBO to stop being ~0 artifacts on AG News/MNLI.

### P0.8 MNLI CF edit-ratio over the editable span (review §8.6c)
`_word_edit_ratio` divides by the full premise+hypothesis length, but the MNLI prompt only allows hypothesis edits — the effective cap varies with premise length and auto-fails legitimate minimal hypothesis edits. When a dataset uses span-restricted CF, compute the ratio over the editable span (hypothesis) only. Keep 0.3 as the cap value; document the denominator in the report.

## P4 — Literature-driven improvements (2024–2026 sweep; each item = citation → concrete change)

1. **CF validity–minimality is now a *replication*, run it as one.** *LLMs Don't Know Their Own Decision Boundaries* ([arXiv:2509.09396](https://arxiv.org/abs/2509.09396)): unconstrained CFs flip but over-edit; minimal CFs under-edit and fail to flip. The smoke run reproduces this exactly (minimal 22% vs free 56% valid). Action: keep both CF variants; report the trade-off per model as a confirmatory result engaging that paper; expect and budget for low minimal-CF coverage on Nova/Qwen (complete-case analysis will shrink — quantify in the pilot before scale-up).
2. **Held-out flip verification is now free.** *Truth or Twist?* ([arXiv:2505.13972](https://arxiv.org/abs/2505.13972)) studies judge-model choice for label-flip evaluation. With 3 models in the run, verify each CF flip with (a) the explained model (construct-defining, keep as primary) and (b) one other model (robustness check breaking the self-verification circularity, review §2.6). One extra classification call per CF.
3. **Reinstate the confidence RQ via verbalized confidence.** Bedrock Converse exposes no logprobs (verified — custom-import InvokeModel only), but the pre-registered confidence–ECS correlation never needed them: elicit verbalized 0–100 confidence / P(true) (Tian et al. 2023; [Xiong et al., ICLR 2024](https://arxiv.org/abs/2306.13063); cf. [arXiv:2603.25052](https://arxiv.org/abs/2603.25052) on the confidence–faithfulness link). One cheap turn per instance restores a whole dropped RQ.
4. **Position against the 2025–26 wave** (review §4 re-check): [arXiv:2602.02639](https://arxiv.org/abs/2602.02639) (*A Positive Case for Faithfulness* — NSG simulatability, 18 models, self > cross-model explanations, 5–15% "egregiously misleading"); [Cross-Examiner, arXiv:2503.08815](https://arxiv.org/abs/2503.08815) (consistency via follow-up questions); [arXiv:2512.07288](https://arxiv.org/abs/2512.07288) (training improves self-explanation faithfulness). The related-work section must say what ECS adds: a *no-logprob, no-followup, single-pass* consistency signal with an erasure bridge.
5. **Add the cross-model agreement analysis the migration makes free.** *Explanation Lottery* ([arXiv:2603.15821](https://arxiv.org/abs/2603.15821)) documents structured cross-model attribution disagreement; [2602.02639](https://arxiv.org/abs/2602.02639) claims privileged self-knowledge. The data to test both at the self-explanation level already exists in every multi-model run: same instance, same strategy, different models. Compare **within-model cross-strategy ECS** vs **cross-model same-strategy agreement** — if within-model consensus ≫ cross-model consensus, that's evidence explanations track model-specific computation (novelty upgrade for Headline positioning); if ≈, explanations are generic task priors. This is arguably the strongest cheap addition available and requires zero new API calls.
6. **CC-SHAP: position, don't benchmark (correction to the earlier demand).** CC-SHAP ([Parcalabescu & Frank 2024](https://arxiv.org/abs/2311.07466)) needs token probabilities → not computable on this study's API surface. State that explicitly as ECS's niche; run CC-SHAP once offline on a small open-weights model over ~30 shared instances as an external-validity anchor if time permits.
7. **Krishna et al. agreement suite + human anchor** (review §4, unchanged): adopt feature-agreement@k as the size-invariant secondary metric (dovetails with P1.2); add an e-SNLI-based human token-importance anchor for the MNLI subset (~20 items).
8. **Salience-weighted null** (review §2.5): add a secondary Monte-Carlo null that samples proportionally to H's salience scores, reported next to the uniform null — bounds the "lift over an unrealistically flat null" objection.
9. **H output schema:** switch `{"salience": {word: score}}` to a list of `[word, score]` pairs (JSON objects silently drop duplicate words, §8.6d); parser accepts both during transition.
