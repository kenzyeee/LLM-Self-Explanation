## 1. Blocking issues

### 1.1 Statistical power is essentially zero — and the apparatus pretends otherwise
- Usable data is **N=10 SST-2 + N=10 MNLI** (AG News N=3 is a smoke test). **Complete cases (all four strategies valid) = 5/23 (22%).**
- The pipeline ships bootstrap CIs (1000×), permutation tests (10,000×), Wilcoxon + Bonferroni, one-sample t-tests, Cohen's d ([`src/statistics/statistical_tests.py`](src/statistics/statistical_tests.py)). At N=5–23 this is machinery with nothing to lift. A bootstrap CI over 23 heterogeneous instances (§2.3) is not a population inference; it is a description of 23 points.
- The study's *own* prior analysis estimates ~150 instances/condition for 80% power ([`pilot_study_analysis.md`](pilot_study_analysis.md) §4.1). Nothing in the current deliverables approaches that. A reviewer cannot distinguish any reported effect from noise.

**This is not a "add more data later" nitpick.** Several headline numbers are computed on N≤5 (e.g., "higher agreement on wrong predictions," N=4) and are reported with interpretation. At this N, report them as anecdotes or not at all.

### 1.2 Single model → every finding is confounded with model identity
- One model (`llama-3.3-70b-versatile`). Madsen et al. (2024, ACL Findings) and Matton et al. (2025, ICLR) both show self-explanation faithfulness/consistency is strongly model-dependent. A single-model result on "do self-explanation *methods* agree" cannot separate *method* effects from *this model's* idiosyncrasies.
- The config (`config/experiment.yaml`) and README advertise a multi-model design but run one. The design doc still lists three (`llama-3.1-8b`, `mixtral-8x7b`, `gemma-7b`) — none of which is the model actually used. See §4.

> **⚠️ RE-CHECK 2026-07-02 — RESOLVED at the design level; evidence-level still OPEN.** The live config now specifies **three distinct non-Anthropic Bedrock families** (Amazon Nova Pro, Qwen3-235B, DeepSeek V3) run **concurrently** per dataset ([`run_experiment.py::_run_model_on_dataset`](scripts/run_experiment.py) via `asyncio.gather`; per-model engine/checkpoint/results, no shared mutable state; dedicated tests in [`tests/test_run_experiment_multimodel.py`](tests/test_run_experiment_multimodel.py)). ### 1.5 The causal-validity bridge is circular and, where run at all, tautology-prone
The intended novelty (Headline #1) is: erase Consensus-Core tokens, show prediction flips more than a random control. Two problems even once it *is* run:
- **Same-model circularity.** CC tokens are extracted from the model's own explanations and the flip is checked with the same model. This measures *self-consistency* (stated evidence ↔ revealed input sensitivity), which the notes now (correctly) concede — but the paper framing must not let "causally validated" drift back toward "faithful." Madsen et al. (2024) and Parcalabescu & Frank (2024, CC-SHAP) are explicit that this is consistency, not faithfulness; the write-up must hold that line everywhere, including the erasure section's own docstring which still says *"causally important tokens."*
- **Thin consensus cores make the test near-tautological.** Mean CC3 = 1.35 tokens, CC4 = 0.87 ([`report.md`](outputs/20260624_131822_acd51a2f/report.md)). When four methods all point at the one obvious cue (`dull`, `misogynistic`), CC = {that cue} and erasing it flips the label — this shows the methods co-locate the salient word, not that "consensus predicts causal importance." The content-word random control helps but is weak at 1-token erasures on ≤20-word inputs. You need inputs with *distributed* evidence for this test to be informative.

> **⚠️ RE-CHECK 2026-07-02 — a third, new blocker for this headline.** Under the multi-model config the erasure pass is **wired to the wrong model**: [`run_validity_tests.py:283`](scripts/run_validity_tests.py) instantiates a single engine from `config.models[0]` (Nova Pro) and re-classifies erased inputs for **every** record in `instance_results.jsonl`, ignoring each record's `model` field. For 2/3 of a multi-model run, a "flip" would be measured on a *different* model than made the original prediction — cross-model transfer, not the same-model stated-vs-revealed sensitivity the construct requires. Headline #1 cannot be computed on any multi-model run until records are grouped by model and re-classified with matching engines (§8.1). No `aggregate_erasure.json` exists for any 2026-07-02 run either — the headline still has zero current-pipeline data.

## 2. Methodology

### 2.1 ECS is Jaccard-based, but the report calls Overlap Coefficient "the primary metric" — pick one
- `compute_ecs` averages **Jaccard** pairs ([`metrics_calculator.py:88–99,143`](src/metrics/metrics_calculator.py)). Every headline ("Mean ECS 0.258", "lift +0.167") is Jaccard.
- Yet the report states *"Overlap Coefficient is the primary pairwise metric… Jaccard is retained as a secondary reference."* The headline construct and the declared-primary construct are different functions. This is an internal inconsistency that a reviewer will flag immediately. If overlap coefficient is primary (a defensible choice given set-size asymmetry), then **ECS itself must be overlap-based**; otherwise the "primary metric" language is misleading.

> **✅ RE-CHECK 2026-07-01 — FIXED (run `20260701_155518_c2494797`).** The internal inconsistency is resolved: the report no longer calls Overlap Coefficient the primary metric. It now states plainly — *"Jaccard is what feeds the headline ECS and ECS-lift (both computed from Jaccard pairwise agreement, never Overlap Coefficient). Overlap Coefficient is reported here as a size-robust complement … for interpreting individual pairs, not for the headline number."* The code matches: `compute_ecs` averages Jaccard pairs ([`metrics_calculator.py:143`](src/metrics/metrics_calculator.py)) built by `compute_pairwise_agreements` → `compute_jaccard_similarity` ([`metrics_calculator.py:88`](src/metrics/metrics_calculator.py)), while Overlap lives in a separate `compute_pairwise_overlaps` ([`metrics_calculator.py:101`](src/metrics/metrics_calculator.py)). Headline metric and declared-primary metric are now the same function. (The set-size ceiling in §2.2 is a separate issue and still stands.)

### 2.2 The Jaccard set-size ceiling is real and partially baked into the *primary composite*
- CF yields **1–2** edited tokens; H/RO yield `dynamic_k = max(3, round(content/5))` ≈ 3. Jaccard(|1|,|3|) ≤ 0.33; Jaccard(|1|,|5|) ≤ 0.20. The `ecs_extraction_perturbation` composite = mean(H–CF, CF–RO) ([`metrics_calculator.py:149–166`](src/metrics/metrics_calculator.py)) is precisely the low-ceiling pair set. Reporting it next to `ecs_extraction_rationale` (higher ceiling) invites the reader to compare two numbers with structurally different maxima.
- Lift-over-chance is the right instinct, but it does not remove the ceiling — it recentres it. A per-pair *normalised* agreement (e.g., overlap coefficient, or Krishna et al.'s feature-agreement@k) is the cleaner fix, and would make the composites comparable.

### 2.3 Pooling heterogeneous datasets into one CI is not valid
- SST-2 mean ECS 0.134, MNLI 0.292, AG News 0.553 — three visibly different distributions. The "Overall N=23" bootstrap CI [0.170, 0.350] pools them. That interval describes a mixture, not a population; its width is driven by between-dataset shifts, not sampling error within a task. Report per-dataset only, or use a mixed model with dataset as a factor — never a pooled CI across tasks with 3–10× mean differences.

### 2.4 ECS is not the same quantity across instances (missingness changes the estimand)
- ECS requires `n_valid ≥ 3` and averages *whichever* cross-paradigm pairs survive. An instance with H,R,RO valid contributes mean(H–R, R–RO) — an **extraction↔rationalisation** number. An instance with H,CF,RO valid contributes mean(H–CF, CF–RO) — an **extraction↔perturbation** number. These are different constructs; pooling them into "Mean ECS" averages apples and oranges. With CF missing in ~48% of instances, the pooled mean is dominated by CF-free instances. The only internally consistent estimand is complete-case ECS (N=5), which is unusable at this N. This is a definitional problem, not just a power problem.

### 2.5 The random baseline models a null the methods do not sample from
- `expected_random_overlap` draws uniform subsets from the input's content vocabulary ([`metrics_calculator.py:7–45`](src/metrics/metrics_calculator.py)). But CF tokens are *edited* words, R tokens are *anchored content words*, H/RO are *top-salience* words — none is a uniform draw. The null is a reasonable first approximation but it **understates chance agreement**, because all four methods are biased toward the same few high-salience tokens for reasons unrelated to "consensus" (e.g., there is literally one sentiment word). A stronger null would resample from a salience-weighted distribution, or permute tokens within matched salience bands. As-is, "lift over chance" is partly lift over an unrealistically flat null.

### 2.6 Self-correction loops inflate coverage and contaminate the evidence sets
- CF gets up to **two** flip-verification attempts with a re-prompt that says *"pick a more impactful word this time"* ([`run_experiment.py:372–419`](scripts/run_experiment.py)); RO gets a hallucination self-correction pass ([`run_experiment.py:428–455`](scripts/run_experiment.py)).
- Consequences a reviewer will raise: (a) the reported CF/RO **coverage and validity rates are for a multi-shot search procedure**, not single-shot elicitation — not comparable to Madsen et al. (2024) or MiCE (Ross et al. 2021), which do not iteratively coach the model; (b) the CF token set becomes partly a function of the *search*, so "which words the model considers critical" is confounded with "which words the correction loop steered it to." At minimum, report single-shot rates alongside, and treat corrected instances as a separate stratum.

> **⚠️ RE-CHECK 2026-07-02 — CF viability collapsed on the new model trio, and the literature now predicts it.** In the Bedrock smoke run, **CF canonical (minimal) validity fell to 22%** (2/9; Nova Pro and Qwen3 produced *zero* valid minimal CFs — they appended clauses instead of editing, breached the 0.3 edit-ratio cap, or emitted multiple JSON objects for multiclass targets), vs. 86% on llama-3.3-70b. This is not (only) a prompt bug: it replicates *LLMs Don't Know Their Own Decision Boundaries* (arXiv:2509.09396, 2025) — frontier LLMs achieve near-perfect flip validity with **excessive** edits when unconstrained, and fail to flip under **minimality** instructions. Implications: (a) at scale, the extraction–perturbation ECS composite and CC4 will be starved of data on 2 of 3 models — quantify per-model CF coverage in the pilot before committing to the 4-strategy design; (b) the pipeline's free-vs-minimal CF contrast becomes a *confirmatory replication* of 2509.09396 and should be reported as such (a genuine, citable result the study gets for free); (c) the choice of flip-verification judge is itself now studied (*Truth or Twist?*, arXiv:2505.13972) — with three models in hand, report both self-verified (construct-defining) and held-out-model-verified flip rates.

### 2.7 Introduced-concept rate (0.75) is confounded with abstractiveness, not just post-hocness
- The rationale prompt *demands* abstractive prose: *"Do not list individual words — write a natural sentence."* Then `parse_rationale` counts content-word lemmas with no input anchor as "introduced" ([`parser.py:219–237`](src/parsing/parser.py)). A fluent one-sentence rationale *necessarily* introduces connective/abstract vocabulary ("implies," "captivating," "dismissive"). So a high introduced-concept rate is largely an artifact of the instruction to be abstractive, not evidence of "post-hoc rationalisation." Atanasova et al. (2023) measure NL-explanation faithfulness with counterfactual and input-reconstruction tests precisely to avoid this surface-overlap confound; a raw anchored-token fraction cannot bear the "rationales are unfaithful" reading the analysis gives it.

### 2.8 Erasure metric is an acknowledged proxy, but the OOD confound is under-treated
- `1 − k/n` first-flip depth ([`redaction_test.py:55–97`](src/metrics/redaction_test.py)) is not ERASER comprehensiveness (DeYoung et al. 2020), which averages predicted-probability drop over k. The code says so honestly. But two deeper issues remain:
  - **OOD inputs.** `[MASK]` on a model never trained with MLM, and `delete` which shortens text, both push inputs off-distribution (Bastings & Filippova 2020, "The elephant in the interpretability room"; Hooker et al. 2019, ROAR). A "flip" may reflect distribution shift, not token importance. Running both operators (ICE 2026, as cited) surfaces the sensitivity but does not resolve which conclusion is correct.
  - **No probability signal.** Because logprobs are unavailable, there is no sufficiency/comprehensiveness curve — only a binary flip at 1–5 tokens. That is a very coarse faithfulness instrument and cannot support fine claims about "tiered" behaviour.

### 2.9 CF insertion-only flips are dropped → systematic bias in the CF evidence set
- `_extract_changed_tokens` only captures `replace`/`delete` opcodes ([`parser.py:279–295`](src/parsing/parser.py)); a flip achieved by inserting "not" yields **no** attributable original token and is discarded as CF-invalid ([`parser.py:270–272`](src/parsing/parser.py)). The memory concedes this drops ~⅔ of minimal MNLI CFs. The result: the surviving CF set is biased toward *substitution*-type edits, which systematically under-represents negation/scope phenomena — exactly the phenomena that matter for NLI. This biases every CF-involving ECS pair on MNLI.

### 2.10 Curation by the assistant introduces selection bias and removes the hard stratum
- The curated "clean-gold" sets were produced by the assistant hand-dropping ~mislabeled/ambiguous items (`curated-datasets-2026-06-23`; DROPS in `scripts/curation_vetting.py`). Two concerns: (a) an LLM/assistant deciding which gold labels are "wrong" can preferentially remove the *hard* cases, inflating classification accuracy (SST-2 100%, and generally high) and plausibly inflating agreement (easy items have one obvious cue → high consensus); (b) this reverses the earlier deliberate inclusion of misclassified instances (`study-redesign-2026-06-17`), so the "consistency ≠ correctness" analysis now rests on only 4 incorrect instances, all from one dataset (§2.11).

### 2.11 The "higher agreement on wrong predictions" claim is fully confounded
- `mean_ecs_incorrect` (N=4) > `mean_ecs_correct` (N=19) is computed pooled across datasets, but **all four incorrect instances are MNLI**, and MNLI has the higher ECS overall. So the contrast is a dataset effect wearing a correctness costume. Either stratify within MNLI (N will be tiny) or drop the claim.

### 2.12 Post-hoc by construction — a conceptual ceiling on what any of this can show
- Classification is a single-shot label with **no** reasoning trace (`classification_*.txt` return `{"label":…}` only). Explanations are then elicited *after* the label, in the same conversation. So every explanation is a reconstruction of a decision the model made in one forward pass without deliberation. This is the Turpin et al. (2023) / Lanham et al. (2023) regime where stated reasons need not track the computation. It is fine to study, but the paper must foreground that **none of these methods can be process-faithful here** — there is no process — so ECS is at best consistency among *post-hoc reconstructions*. The current framing gestures at this but the metric names ("faithfulness" in `redaction_test`, "validity") keep leaking the stronger reading.

### 2.13 Minor-but-real metric issues
- **Kendall's τ requires ≥4 overlapping tokens** ([`metrics_calculator.py:78`](src/metrics/metrics_calculator.py)); with k≈3 it is `None` on most instances, so "Kendall τ(H,RO)=0.33" rests on a handful of instances. RBO(p=0.9) is top-weighted but reported as a single mean without its own CI.
- **`assign_implicit_ranks`** treats H's salience order and RO's list order as comparable ranks, but H is a *graded* score truncated to top-k while RO is an *ordinal* elicitation — τ between them mixes two rank semantics.
- **CC4 for 3-valid instances is redefined as "intersection of the 3 present"** ([`run_experiment.py:684–689`](scripts/run_experiment.py)), so "CC4" does not always mean "all four methods." The label CC4 is overloaded.

## 3. Prompt-by-prompt critique (with literature comparison)

General issue across all elicitation prompts: they are sent **as a follow-up turn after the label**, with the classification prompt + the model's label in context ([`run_experiment.py:291–295`](scripts/run_experiment.py)). Every explanation is thus conditioned on the committed label — maximally post-hoc (see §2.12). Also, because the input text is re-quoted in each explanation prompt *and* present in the classification turn, task-framing tokens (for MNLI, the literal `Premise:`/`Hypothesis:` scaffold) are in the model's context; leakage into evidence sets is currently caught only because `DISCOURSE_WORDS` now lists `premise/hypothesis/entail/contradict` ([`normalizer.py:11–26`](src/normalization/normalizer.py)). That is a denylist patch, not a structural fix — any unforeseen framing token (e.g. `relationship`, `category`, `topic`) would leak.

### 3.1 Classification (`classification_sst2/mnli/ag_news.txt`)
- Label-only JSON, no CoT. **Good** and consistent with Madsen et al. (2024) using label-only prediction. But three per-dataset promptings differ in framing verbs ("Analyze the sentiment" / "Determine the logical relationship" / "Categorize"), which is a small confound if you ever compare *across* datasets — the elicitation register is not held constant.
- No confidence elicited (dropped). Kadavath et al. (2022) and Tian et al. (2023) show verbalised confidence is obtainable without logprobs; abandoning the confidence RQ entirely (rather than using verbalised P(True)) throws away a pre-registered axis that needed no logprobs.

### 3.2 Highlighting (`highlighting_explain.txt`)
- Terse per-word 1–10 salience. Reasonable and close to self-generated feature attribution (Huang et al. 2023, arXiv:2310.11207). **But**: (a) grading *every* word then truncating to `dynamic_k` discards the score magnitudes the prompt worked to elicit — you collect a full salience vector and throw away everything but a top-k **set** before ECS. Weighted-Jaccard or rank-correlation on the full vector is the obvious, cheap improvement (also raised in [`pilot_study_analysis.md`](pilot_study_analysis.md) §4.3). (b) The committed run used a *different, longer* highlighting prompt than the file now on disk (§1.3), so the reported H behaviour is not what this file produces.
- Faithfulness of such self-attributions should be checked with sufficiency/comprehensiveness (DeYoung et al. 2020), not only cross-method overlap.

### 3.3 Rationale (`rationale_explain.txt`)
- *"In one sentence, explain why…"* then *"Do not list individual words."* This deliberately forces abstractive prose and then the pipeline extracts anchored content words from it — a mismatch between what is elicited (a narrative) and what is measured (token overlap). The introduced-concept metric is confounded by this instruction (§2.7). e-SNLI (Camburu et al. 2018) and Wiegreffe et al. (2021) treat free-text rationales as their own object with dedicated faithfulness tests; reducing a rationale to an anchored-token bag to force it into a Jaccard with a word-list is the weakest link in the ECS construction.

### 3.4 Counterfactual (`counterfactual_explain.txt`, `_multiclass_mnli`, `_free`)
- Minimal-edit framing (*"Make as few edits as possible"*) follows MiCE (Ross et al. 2021) and the Madsen et al. (2024) counterfactual self-test; edits/length minimality matches MiCE's metric — **good**. MNLI hypothesis-only editing to avoid segment-confounds is a genuinely thoughtful touch ([`counterfactual_explain_multiclass_mnli.txt`](prompts/counterfactual_explain_multiclass_mnli.txt)).
- Problems: (a) self-generated + self-verified + **coached over two attempts** (§2.6) departs from single-shot elicitation and from search-based editors, so CF validity numbers are not comparable to those baselines; (b) insertion-only flips discarded (§2.9) bias the attribution; (c) the *committed* CF prompt (verbose, with a `{"rewritten": null}` escape hatch) differs from the terse disk version — again, results ≠ code.

### 3.5 Rank Ordering (`rank_ordering_explain.txt`)
- *"3–5 most important words, ranked."* Same extraction paradigm as H (correctly excluded from ECS). Fine. But H and RO are elicited as *separate* calls that both essentially ask "what are the important words," so their high agreement (overlap 0.80) is close to a **test–retest reliability** of one method under two surface forms, not independent corroboration. That is worth stating explicitly; it also means the "extraction paradigm" is over-represented (2 of 4 methods), pulling several ECS pairs toward the easy case.

### 3.6 Orphaned / unused prompt assets
- `prompts/` contains `*_alt.txt` variants, `highlighting_k2/k5.txt`, and `confidence_calibration.txt`. The config advertises prompt-wording robustness, top-k, and normalization ablations (Req 14–17). None of these ablations appear in the committed run's outputs. Shipping the prompts without the ablation results is scope theatre — either run them or remove the claims. `confidence_calibration.txt` is dead (confidence dropped).

## 4. Positioning & novelty vs. related work

- **The framing is a known result.** "Different explanation methods disagree" is Krishna et al. (2022/2023, *The Disagreement Problem in Explainable ML*); the study's own notes call the bare "agreement ≠ faithfulness" point "semi-obvious." Applying it to a single model's *self*-explanations is a reasonable delta, but Madsen et al. (2024), Parcalabescu & Frank (2024, CC-SHAP), Turpin et al. (2023), and Matton et al. (2025) already probe self-explanation consistency/faithfulness. The incremental contribution must therefore be the two headline analyses — which currently have no data (§0).
- **No baseline comparison.** Krishna et al. already define feature-agreement, rank-agreement, and sign-agreement metrics; ECS reinvents feature/rank agreement without citing or benchmarking against them. CC-SHAP (Parcalabescu & Frank 2024) is a directly competing self-consistency measure that needs no input editing (avoiding the OOD problem in §2.8) and gives a continuous score. A paper introducing ECS must show what ECS buys over CC-SHAP and over Krishna's agreement suite. None is reported.
- **No human/reference anchor.** With no human token-importance annotations (even 20 items × 3 raters), low ECS cannot be attributed to "methods disagree" vs. "methods agree on the wrong tokens." e-SNLI provides human NLI rationales off the shelf for exactly this.
- **The design doc's model list is not even the model used** (`llama-3.1-8b`/`mixtral`/`gemma` vs. `llama-3.3-70b`), so the "related work / methodology" a reader would see is doubly wrong.

> **⚠️ RE-CHECK 2026-07-02 — positioning must absorb the 2025–26 wave; the niche is still open but closing.** A fresh sweep (through July 2026) finds no published paper doing exactly *within-model, cross-strategy agreement of self-explanations + an erasure bridge on API-only models* — the delta survives. But four lines of work now border it and must be engaged: (1) **arXiv:2602.02639** (*A Positive Case for Faithfulness*, Feb 2026) evaluates self-explanations on 18 frontier models via **Normalized Simulatability Gain** and finds self-explanations beat stronger external models' explanations ("privileged self-knowledge") while 5–15% remain "egregiously misleading" — ECS must argue what token-overlap consensus measures that simulatability does not; (2) **Cross-Examiner** (arXiv:2503.08815) probes explanation consistency via follow-up questions — a competing consistency operationalisation; (3) the **"Explanation Lottery"** result (arXiv:2603.15821) documents structured cross-*model* attribution disagreement — the new 3-model design can and should speak to this with a cross-model agreement analysis it currently doesn't run; (4) **arXiv:2512.07288** (Dec 2025) shows training/finetuning shifts self-explanation faithfulness — a moderator for the discussion. On the CC-SHAP benchmark demand above, one correction from the migration: CC-SHAP **requires token probabilities**, which Bedrock's Converse API does not expose for any of the three models (verified) — so CC-SHAP is *not computable* in this study's API-only setting. That converts the demand from "benchmark against CC-SHAP" into "state precisely that ECS's niche is the no-logprob regime, and benchmark against CC-SHAP once, offline, on one open-weights model as an external-validity check."

## 8. New defects found in the 2026-07-02 re-check

These were found by cross-examining the multi-model smoke run ([`20260702_144026_c22695ee`](outputs/20260702_144026_c22695ee/report.md)) against the raw `instance_results.jsonl` and the current code. §8.2–8.4 are not migration artifacts — they corrupt the ECS numbers of **every run to date**, including the Groq-era results this review previously discussed.

### 8.1 Erasure pass re-classifies with the wrong model under multi-model (BLOCKING)
[`run_validity_tests.py:283`](scripts/run_validity_tests.py) builds one engine from `config.models[0]` and uses it for every record, ignoring each record's `model` field. A Qwen3 or DeepSeek prediction would have its erasure flip checked by **Nova Pro** — cross-model transfer, not the same-model stated-vs-revealed sensitivity that defines Headline #1. Fix: group records by `model`, instantiate a matching engine per group. (Also: still no `aggregate_erasure.json` for any current run.)

### 8.2 R-evidence is lemmas; H/RO/CF evidence is surface forms — with lemmatization OFF, they can never match (BLOCKING for the ECS construct)
`parse_rationale` extracts **spaCy lemmas** ([`parser.py:207`](src/parsing/parser.py)); H and RO sets are **surface forms** passed through `normalize_tokens`, which with the live `use_lemmatization: false` does not lemmatize. Nothing reconciles the two spaces. Live proof from the smoke run (SST-2, all three models): H = `{ice, moved, scenes, tears}`, R = `{move, scene}` → reported **H–R Jaccard = 0.0000** despite 2/4 morphological overlap. Every R-involving pair (H–R, R–CF, R–RO), the extraction–rationale composite, ECS itself, and the introduced-concept rate are deflated by an inflection-dependent amount — i.e., **dataset-confounded measurement error in the headline metric**. This bug was introduced when v2.0 turned evidence lemmatization off (the anchoring lemmatizer stayed on, masking the problem in `is_anchored`). Fix: compare all evidence in one canonical space — either lemmatize evidence sets (turn `use_lemmatization` back on), or have R return the **matched input surface forms** its anchor already found, or make the Jaccard anchor-lemma-aware. Add a fixture test: two sets differing only by inflection must score 1.0.

### 8.3 CF evidence bypasses `normalize_tokens` entirely (MAJOR)
[`run_experiment.py:420`](scripts/run_experiment.py) stores raw difflib `from_tokens` (lowercased, punctuation-stripped only). CF sets therefore retain **stopwords and unlemmatized forms** that H/R/RO sets can never contain (smoke-run example: CF = `{happened, have, shouldn't}`). Consequences: CF-pair Jaccard unions are inflated by unmatchable tokens; **R–CF = 0.0000 across the entire smoke run** (double-hit with §8.2); and the Monte-Carlo null — which draws from the *content-word* vocabulary — mis-models CF sets that live partly outside that vocabulary, so "lift over chance" is mis-centred for every CF pair. Fix: pass CF tokens through the same normalization as the other strategies (with §8.4's negation carve-out), and align the null's vocabulary model.

### 8.4 Contracted negations are silently excluded from evidence — on an NLI task (MAJOR)
`POLARITY_WORDS` ([`parser.py:30`](src/parsing/parser.py), [`normalizer.py:31`](src/normalization/normalizer.py)) whitelists bare negations (`no, not, never…`) but NLTK's stopword list also contains **contracted negations** (`shouldn't, don't, isn't, won't…`), which are therefore dropped from H, R, and RO evidence — while CF keeps them (§8.3). Smoke-run consequence: DeepSeek's MNLI rank-ordering returned `["Iraq", "shouldn't", "happened"]`; `shouldn't` was discarded as a stopword, leaving 2 < 3 valid tokens, and **the entire RO strategy was invalidated** — on the one instance where negation was the label-critical evidence. For MNLI, the pipeline systematically excludes the most causally relevant token class from the consensus construct. Fix: extend the polarity whitelist to contracted negations (and their tokenization variants) symmetrically across all four extraction paths.

### 8.5 Report renders wrong-prediction instances as "SKIPPED — no explanation strategies elicited, no ECS computed" — but they were elicited and are in the aggregates (MAJOR, report integrity)
[`data_models.py:773`](src/utils/data_models.py) prints the SKIPPED banner for any `correct == False` instance — stale logic from the era when wrong predictions were short-circuited. Since D5, the pipeline elicits all four strategies for misclassifications ([`run_experiment.py:207`](scripts/run_experiment.py)) and computes their ECS. In the smoke report the same DeepSeek MNLI instance appears **both** as "SKIPPED… no ECS computed" (details) **and** as a per-dataset row with ECS 0.233 and full H/R/CF coverage, feeding "Mean ECS — incorrect preds". The auto-generated artifact reviewers read first contradicts the data it summarizes, and the incorrect-prediction stratum's evidence is invisible. Fix: render full details with a "wrong prediction — included per D5" tag; reserve SKIPPED for the unparseable-classification case that actually skips.

### 8.6 Smaller but real (grouped)
- **Clustered pooling.** With 3 model-rows per instance, the pooled "Overall" bootstrap CI treats correlated rows as independent → CI too narrow. Report per-model×dataset cells; if pooling, cluster-bootstrap by instance or use a mixed model (instance random effect, model fixed effect). Compounds §2.3.
- **Rank-agreement artifact.** Kendall τ / RBO compare **raw-case** H keys (`h_ordered`: "NFC", "Iraq") against **normalized** RO tokens ([`run_experiment.py:498–507`](scripts/run_experiment.py)) — case mismatches zero them out on capitalized datasets: smoke-run RBO(H,RO) = 0.0000/0.0000/0.0076 on AG News and τ = None everywhere, while the set-based H–RO overlap coefficient is 0.72. The "H and RO barely rank-agree" reading is an artifact. Normalize both rank lists identically.
- **MNLI CF cap denominator.** The 0.3 edit-ratio is computed over the **full premise+hypothesis** text, but the MNLI CF prompt (correctly) allows editing only the hypothesis — the *effective* cap on the editable span varies with premise length (a full-hypothesis rewrite ≈ 0.5 ratio on the smoke instance → auto-fail). Normalize the ratio by the editable span.
- **H salience schema loses duplicates.** `{"salience": {word: score}}` is a JSON object; repeated words ("Iraq" twice) collapse to the last occurrence silently. A list-of-pairs schema would preserve them.
- **Inert, unseeded randomization.** `random.shuffle(strategy_order)` ([`run_experiment.py:245`](scripts/run_experiment.py)) is unseeded (non-reproducible order) and scientifically inert — each strategy is an independent 3-message call, so order cannot affect the model. The comment implies an order-effect control that doesn't exist. Delete or seed it; don't imply a control.
- **Stale internal tooling/doc noise.** [`.audit_workflow.js`](.audit_workflow.js) CONTEXT still says "single LLM llama-3.3-70b via Groq" and references the deleted `validity_checker.py`; `.env.example` advertises Anthropic Claude availability in eu-north-1 while the config documents Anthropic as dropped; `inference_engine.py`'s default model id is a US-profile Llama that would not resolve in the documented EU region (harmless — always overridden — but misleading).
